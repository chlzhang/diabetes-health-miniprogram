# 糖前期生活监督小程序 — 产品需求规格书

> 版本：v1.0 MVP  
> 日期：2026-06-02  
> 目标用户：糖前期人群 + 家庭成员

---

## 一、产品概述

### 1.1 背景
糖前期（空腹血糖受损 5.6~6.9 mmol/L，或糖耐量异常）是糖尿病的前兆阶段。通过饮食控制、规律运动和血糖监测，可以有效延缓或逆转进展为 2 型糖尿病。目前缺乏一款面向家庭、便于每日打卡监督的轻量工具。

### 1.2 产品定位
微信小程序，面向糖前期患者及其家庭成员，提供「饮食记录 + 运动记录 + 血糖打卡 + 家庭互助」四大核心功能，通过每日打卡和家人监督建立健康习惯。

### 1.3 成功指标（MVP 阶段）
| 指标 | 目标 |
|------|------|
| 日活打卡率 | ≥ 70%（连续 7 天） |
| 每日记录次数 | 平均 ≥ 2 条（饮食 + 血糖） |
| 家庭组创建率 | ≥ 60% 用户加入家庭组 |

---

## 二、用户角色

| 角色 | 说明 |
|------|------|
| 患者 | 糖前期人群，主要记录者，记录饮食、运动、血糖 |
| 家属 | 家庭成员，查看患者记录，提供监督和鼓励 |

> MVP 阶段两者使用同一套界面，仅通过「今日记录状态」区分（记录者 / 查看者）。

## 三、功能模块详细设计

### 3.1 饮食记录

#### 3.1.1 数据字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| meal_type | enum | ✅ | breakfast / lunch / dinner / snack |
| record_time | datetime | ✅ | 记录时间，默认当前时间 |
| description | string | — | 文字描述 |
| main_food | string | — | 主食类型（米饭/面条/馒头/杂粮等） |
| main_food_amount | number | — | 主食份量（克） |
| has_vegetable | boolean | — | 是否有蔬菜 |
| has_protein | boolean | — | 是否有优质蛋白 |
| has_sweet_food | boolean | — | 是否有甜食 |
| has_sweet_drink | boolean | — | 是否有含糖饮料 |
| photo_url | string | — | 餐食照片 URL |
| health_level | enum | auto | good / ok / warn |

#### 3.1.2 健康评级规则

- good：主食 ≤150g，有蔬菜，有蛋白质，无甜食，无含糖饮料
- ok：主食 ≤200g，有蔬菜或蛋白质
- warn：其他情况

### 3.2 运动记录

#### 3.2.1 数据字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sport_type | string | ✅ | 运动类型 ID |
| duration_min | number | ✅ | 时长（分钟） |
| intensity | enum | ✅ | light / moderate / vigorous |
| calories | number | auto | 估算消耗（千卡） |
| note | string | — | 备注 |

#### 3.2.2 运动类型预设

| 类型 | 默认强度 | MET |
|------|---------|-----|
| 快走 | moderate | 3.5 |
| 慢跑 | moderate | 7.0 |
| 骑车 | moderate | 6.0 |
| 游泳 | moderate | 6.0 |
| 瑜伽 | light | 2.5 |
| 力量训练 | vigorous | 5.0 |
| 太极 | light | 2.5 |
| 广场舞 | moderate | 4.0 |

消耗公式：MET × 体重(kg) × 时长(h)

#### 3.2.3 每周目标
默认：每周 ≥ 150 分钟中等强度运动，首页进度条显示

### 3.3 血糖打卡

#### 3.3.1 数据字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| time_point | enum | ✅ | fasting / post_meal / bedtime |
| blood_sugar | number | ✅ | 血糖值 mmol/L |
| note | string | — | 备注 |

#### 3.3.2 血糖标准

| 时段 | 正常 | 偏低 | 偏高 |
|------|------|------|------|
| 空腹 | < 6.1 | < 3.9 | ≥ 6.1 |
| 餐后2h | < 7.8 | < 3.9 | ≥ 7.8 |
| 睡前 | < 7.0 | < 3.9 | ≥ 7.0 |

### 3.4 家庭互助

- 创建家庭组：生成 6 位邀请码，有效期 7 天
- 加入家庭组：输入邀请码
- 家庭组上限：8 人
- 家庭看板：显示每位成员今日打卡状态
- 点赞互动：可点赞记录，对方收到通知

### 3.5 首页看板

- 今日 3 餐状态（✓/✗）+ 健康评级
- 本周运动时长进度条
- 最近血糖值 + 状态标记
- 连续打卡天数
- 家庭成员今日状态小圆点

## 四、数据库设计（云开发）

**users**：openid、昵称、头像、身高、体重、年龄、family_id  
**families**：名称、邀请码、过期时间、创建者、成员列表  
**meals**：user_id、餐次、时间、描述、主食信息、蔬菜/蛋白/甜食标记、照片URL、健康评级  
**exercises**：user_id、运动类型、时长、强度、消耗、时间  
**blood_sugars**：user_id、时段、血糖值、状态、时间  
**likes**：点赞者、目标类型、目标ID

## 七、验收标准

- [ ] 饮食：选择餐次 → 记录 → 自动评级 → 首页状态更新
- [ ] 运动：选择类型+时长 → 估算消耗 → 周进度条累计
- [ ] 血糖：选时段+输入值 → 自动判定 → 异常提示
- [ ] 家庭：创建/加入 → 看板显示成员状态 → 点赞可用
- [ ] 首页：所有数据正确聚合显示
